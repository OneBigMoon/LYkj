## UML部署图绘制指南

### 图例说明

#### 节点类型（Node）
- **<<device>>** - 物理设备（服务器、工作站等）
- **<<execution environment>>** - 执行环境（容器、虚拟机等）
- **<<artifact>>** - 制品（应用程序、文件等）

#### 连接类型
- **实线** - 物理连接（网络连接）
- **虚线** - 依赖关系
- **<<deploy>>** - 部署关系

## 2. 亿图图示绘制步骤

### 步骤1：创建新的UML部署图
1. 打开亿图图示
2. 选择"软件和数据库" → "UML建模" → "部署图"
3. 创建新文档

### 步骤2：添加节点（Node）
从左侧图形库拖拽以下元素：

#### 设备节点（Device）
- 图形：立方体（3D盒子）
- 标注：`<<device>>`
- 命名：服务器名称 + 配置

**示例：**
```
┌──────────────────┐
│  <<device>>      │
│  Web服务器-1     │
│  (WEB-01)        │
│  16C32G          │
└──────────────────┘
```

#### 执行环境（Execution Environment）
- 图形：立方体（3D盒子）
- 标注：`<<execution environment>>`
- 命名：Kubernetes集群、Docker等

**示例：**
```
┌──────────────────┐
│<<execution env>> │
│ Kubernetes集群   │
│  (K8S-PROD)      │
└──────────────────┘
```

#### 制品（Artifact）
- 图形：文档图标或矩形
- 标注：`<<artifact>>`
- 命名：应用程序名称

**示例：**
```
┌────────────┐
│<<artifact>>│
│ Web前端    │
│ React 18   │
└────────────┘
```

### 步骤3：添加连接关系

#### 网络连接（实线）
- 类型：实线
- 标注：协议（HTTP、TCP等）
- 带宽：可选标注

#### 部署关系（虚线箭头）
- 类型：虚线箭头
- 标注：`<<deploy>>`
- 方向：从制品指向节点

#### 依赖关系（虚线箭头）
- 类型：虚线箭头
- 标注：`<<use>>`
- 方向：从使用方指向被使用方

### 步骤4：添加分组容器
使用"泳道"或"容器"对不同网络区域分组：
- DMZ区域
- 应用区域
- 业务区域
- 数据区域
- 计算区域

## 4. 网络连接关系

### 4.1 外部访问路径
```
Internet → WAF → 负载均衡器 → Web服务器 → API服务器 → 业务服务
```

### 4.2 内部服务调用
```
业务服务 → 数据库（读写分离）
业务服务 → 缓存（Redis）
业务服务 → 消息队列（Kafka）
业务服务 → GPU集群（训练任务）
业务服务 → 存储集群（对象存储）
```

### 4.3 数据同步
```
主数据库 → 从数据库1（主从复制）
主数据库 → 从数据库2（主从复制）
Redis主 → Redis从（主从复制）
```

## 5. 部署关系说明

### 5.1 Web服务器部署
- **节点**: WEB-01, WEB-02, WEB-03
- **制品**: React 18前端应用 + Nginx
- **关系**: `<<deploy>>`
- **说明**: 前端应用部署在Nginx中，通过负载均衡器分发流量

### 5.2 API服务器部署
- **节点**: API-01, API-02, API-03
- **制品**: FastAPI应用 + Python 3.9
- **关系**: `<<deploy>>`
- **说明**: API服务部署在Python运行环境中，提供RESTful接口

### 5.3 Kubernetes集群部署
- **节点**: K8S集群（Master×3 + Worker×5）
- **制品**: 微服务容器（Pod）
- **关系**: `<<deploy>>`
- **说明**: 所有业务微服务以容器形式部署在K8S集群中

### 5.4 数据库部署
- **节点**: DB-01 (Master), DB-02/03 (Slave)
- **制品**: MySQL 8.0
- **关系**: `<<deploy>>`
- **说明**: 主从架构，读写分离，主库负责写入，从库负责读取

### 5.5 GPU集群部署
- **节点**: GPU-01~05
- **制品**: PyTorch 1.12 / TensorFlow 2.9
- **关系**: `<<deploy>>`
- **说明**: AI训练框架部署在GPU服务器上，支持分布式训练

## 总结

本UML部署图展示了AI训练中台的完整物理部署架构，包括：

1. **5个网络区域**: DMZ、应用、业务、数据、计算
2. **40+台物理服务器**: 涵盖Web、API、数据库、GPU等
3. **Kubernetes集群**: 8节点集群，部署6类微服务
4. **高可用架构**: 负载均衡、主从复制、多副本部署
5. **清晰的网络隔离**: 不同区域通过防火墙和ACL隔离

该部署架构支持千万级用户并发访问，满足企业级AI训练平台的所有需求。
