# 细节优化方案

## 📋 优化目标

将硬编码的指标数据改为从VirtualDB动态计算，提升系统的真实感和一致性。

### 1. 智能指挥中心
- ✅ **训练任务数** - 从VirtualDB.trainTasks计算
- ✅ **在线用户数** - 从VirtualDB.users计算（status='在线'）
- ⚠️ **模型准确率** - 可从VirtualDB.models平均计算
- ⚠️ **数据资产** - 保持静态（演示数据）

### 2. AI模型工厂
- ✅ **训练任务数** - 从VirtualDB.trainTasks计算

- ✅ **标注任务数** - 从VirtualDB.labelTasks计算
- ✅ **进行中任务** - 从VirtualDB.labelTasks筛选（status='进行中'）
- ⚠️ **AI标注率** - 可计算平均值

### 4. 审计中心
- ✅ **今日操作数** - 从VirtualDB.auditLogs计算（今日）
- ✅ **活跃用户数** - 从VirtualDB.users计算（status='在线'）
- ✅ **高风险操作** - 从VirtualDB.auditLogs筛选（risk='高风险'）

## 💡 优化策略

### 策略A: 完全动态化（推荐）
**优点**:
- 数据完全真实
- 与VirtualDB完全同步
- 用户操作后立即反映

**缺点**:
- 初始数据较少，指标可能不够丰富
- 需要添加更多初始数据

**实现**:
```javascript
// 在页面onMount时计算
onMount() {
  updateDashboardMetrics();
}

function updateDashboardMetrics() {
  const tasks = VirtualDB.getTable('trainTasks');
  const users = VirtualDB.getTable('users');
  const models = VirtualDB.getTable('models');
  
  // 更新训练任务数
  document.getElementById('taskCount').textContent = tasks.length;
  
  // 更新运行中任务数
  const runningTasks = tasks.filter(t => t.status === '运行中').length;
  document.getElementById('runningTasks').textContent = runningTasks;
  
  // 更新在线用户数
  const onlineUsers = users.filter(u => u.status === '在线').length;
  document.getElementById('onlineUsers').textContent = onlineUsers;
  
  // 更新模型准确率（平均值）
  if (models.length > 0) {
    const avgAccuracy = models.reduce((sum, m) => sum + m.accuracy, 0) / models.length;
    document.getElementById('accuracy').textContent = avgAccuracy.toFixed(1) + '%';
  }
}
```

### 策略B: 混合模式（当前）
**优点**:
- 保持丰富的演示数据
- 关键指标动态化
- 平衡真实性和展示效果

**缺点**:
- 部分数据不一致
- 需要维护两套数据

**实现**:
- 核心指标（用户、任务、模型）动态化
- 辅助指标（数据资产、QPS等）保持静态

### 策略C: 保持静态（不推荐）
**优点**:
- 数据丰富
- 无需计算

**缺点**:
- 与实际操作不一致
- 缺乏真实感

## ⚠️ 注意事项

### 1. 性能考虑
- 指标计算应该高效
- 避免频繁的DOM操作
- 使用防抖/节流

### 2. 用户体验
- 指标更新应该平滑
- 可以添加数字滚动动画
- 避免突兀的变化

### 3. 数据一致性
- 确保所有指标来源一致
- 定期校验数据准确性
- 处理边界情况（空数据）
