## 📊 数据库结构

### 表结构
```javascript
{
  users: [],          // 用户表
  trainTasks: [],     // 训练任务表
  experiments: [],    // 实验表（预留）
  labelTasks: [],     // 标注任务表（预留）
  models: [],         // 模型表
  auditLogs: [],      // 审计日志表
  nextId: {           // 自增ID管理
    users: 4,
    trainTasks: 4,
    experiments: 3,
    labelTasks: 3,
    models: 3,
    auditLogs: 1
  }
}
```

### 默认数据
- 用户：3个
- 训练任务：3个（运行中、排队中、已完成各1个）
- 实验：2个
- 标注任务：2个
- 模型：2个
- 审计日志：空（操作后自动生成）

## 💡 技术亮点

### 1. 完整的CRUD操作
- Create: 添加用户、创建训练任务
- Read: 动态渲染列表
- Update: 编辑用户、更新任务状态
- Delete: 删除用户、删除任务

### 2. 状态机管理
训练任务状态流转：
```
排队中 → 运行中 → 已完成 → 注册模型
         ↓
       已暂停 → 继续 → 运行中
         ↓
       已停止
```

### 3. 自动审计
所有操作自动记录，包括：
- 操作类型
- 操作目标
- 详细信息
- 风险等级
- 操作人
- IP地址
- 时间戳

### 4. 智能UI更新
- 根据状态显示不同按钮
- 根据状态显示不同颜色
- 根据状态显示不同信息

### 5. 数据关联
- 训练任务 → 模型（注册）
- 操作 → 审计日志（自动记录）
