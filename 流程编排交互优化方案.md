# 流程编排交互优化方案

## 🎯 问题分析

### 当前问题
1. ❌ 画布和列表同时显示，界面混乱
2. ❌ 点击"编辑"按钮没有明确的交互逻辑
3. ❌ 用户不知道如何进入编辑模式
4. ❌ 画布一直占用空间，即使不需要编辑

### 用户期望
1. ✅ 默认显示流程列表（主界面）
2. ✅ 点击"编辑"进入该流程的画布编辑模式
3. ✅ 点击"新建流程"进入空白画布创建模式
4. ✅ 编辑模式下有"返回列表"按钮

---

## 💡 优化方案

### 方案A: 双视图切换（推荐）⭐

#### 视图1: 列表模式（默认）
```
┌─────────────────────────────────────┐
│ 📋 流程列表                          │
│ [搜索] [筛选] [+ 新建流程]          │
├─────────────────────────────────────┤
│ 流程名称 | 类型 | 节点数 | 状态 | 操作│
│ FLOW-101 | ETL  | 12    | 运行中| [编辑]│
│ FLOW-102 | 训练 | 8     | 完成  | [编辑]│
└─────────────────────────────────────┘
```

#### 视图2: 编辑模式
```
┌─────────────────────────────────────┐
│ 🔄 流程编排: FLOW-101                │
│ [← 返回列表] [保存] [运行] [清空]   │
├──────────┬──────────────────────────┤
│ 节点库   │      画布区域            │
│ 📊数据源 │                          │
│ ⚙️处理   │    [拖拽节点到此处]      │
│ 🎯训练   │                          │
│ 🚀部署   │                          │
└──────────┴──────────────────────────┘
```

#### 交互流程
1. 用户打开流程编排页面 → 显示列表模式
2. 点击"新建流程" → 切换到编辑模式（空白画布）
3. 点击"编辑"按钮 → 切换到编辑模式（加载该流程）
4. 点击"返回列表" → 切换回列表模式

---

### 方案B: 模态框编辑

#### 优点
- 不改变主界面结构
- 编辑器独立弹窗

#### 缺点
- 画布空间受限
- 不适合复杂流程编辑

---

### 方案C: 标签页切换

#### 优点
- 可以同时打开多个流程编辑

#### 缺点
- 实现复杂
- 可能造成混乱

---

### 2. JavaScript函数

```javascript
// 进入流程编辑器
function enterFlowEditor(flowId) {
  // 隐藏列表视图
  document.getElementById('flowListView').style.display = 'none';
  
  // 显示编辑器视图
  document.getElementById('flowEditorView').style.display = 'block';
  
  if (flowId) {
    // 加载现有流程
    loadFlowData(flowId);
    document.getElementById('currentFlowName').textContent = flowId;
  } else {
    // 新建流程
    clearCanvas();
    document.getElementById('currentFlowName').textContent = '新建流程';
  }
  
  // 初始化画布
  initFlowDesigner();
  
  showNotification(`📝 进入编辑模式`, 'info');
}

// 退出流程编辑器
function exitFlowEditor() {
  // 显示列表视图
  document.getElementById('flowListView').style.display = 'block';
  
  // 隐藏编辑器视图
  document.getElementById('flowEditorView').style.display = 'none';
  
  showNotification(`✓ 已返回流程列表`, 'success');
}

// 加载流程数据
function loadFlowData(flowId) {
  // 从VirtualDB或模拟数据加载流程
  showNotification(`📂 正在加载流程: ${flowId}...`, 'info');
  
  // 模拟加载节点和连接
  setTimeout(() => {
    showNotification(`✓ 流程加载完成`, 'success');
  }, 500);
}

// 保存流程
function saveFlow() {
  const flowName = document.getElementById('currentFlowName').textContent;
  
  if (flowName === '新建流程') {
    // 弹窗输入流程名称
    openModal('保存流程', `
      <div style="display: flex; flex-direction: column; gap: 16px;">
        <div>
          <label style="display: block; margin-bottom: 4px; font-size: 13px;">流程名称</label>
          <input id="newFlowName" type="text" placeholder="请输入流程名称" style="width: 100%; padding: 8px; background: rgba(51, 65, 85, 0.5); border: 1px solid #334155; border-radius: 6px; color: #E2E8F0;">
        </div>
        <div>
          <label style="display: block; margin-bottom: 4px; font-size: 13px;">流程类型</label>
          <select id="newFlowType" style="width: 100%; padding: 8px; background: rgba(51, 65, 85, 0.5); border: 1px solid #334155; border-radius: 6px; color: #E2E8F0;">
            <option>ETL流程</option>
            <option>训练流程</option>
            <option>推理流程</option>
            <option>数据处理</option>
            <option>模型评估</option>
            <option>自动部署</option>
          </select>
        </div>
        <button class="btn btn-primary" onclick="confirmSaveFlow()">保存</button>
      </div>
    `);
  } else {
    // 直接保存
    showNotification(`✓ 流程 ${flowName} 已保存`, 'success');
  }
}

// 确认保存新流程
function confirmSaveFlow() {
  const name = document.getElementById('newFlowName').value;
  const type = document.getElementById('newFlowType').value;
  
  if (!name) {
    showNotification('⚠️ 请输入流程名称', 'warning');
    return;
  }
  
  // 保存到VirtualDB
  const flowId = 'FLOW-' + (Math.floor(Math.random() * 900) + 100);
  document.getElementById('currentFlowName').textContent = flowId;
  
  closeModal();
  showNotification(`✓ 流程 ${flowId} 创建成功`, 'success');
}

// 运行流程
function runFlow() {
  const flowName = document.getElementById('currentFlowName').textContent;
  showNotification(`▶️ 正在运行流程: ${flowName}...`, 'info');
  
  setTimeout(() => {
    showNotification(`✓ 流程执行完成`, 'success');
  }, 2000);
}

// 清空画布
function clearCanvas() {
  if (confirm('确定要清空画布吗？')) {
    // 清空所有节点和连接
    document.querySelectorAll('.flow-canvas-node').forEach(node => node.remove());
    document.querySelectorAll('#flowConnections line, #flowConnections polygon').forEach(el => el.remove());
    
    // 显示占位符
    const placeholder = document.getElementById('canvasPlaceholder');
    if (placeholder) placeholder.style.display = 'block';
    
    showNotification('✓ 画布已清空', 'success');
  }
}
```

## 🎯 用户体验提升

### 1. 清晰的视图切换
- 列表模式：专注于浏览和管理流程
- 编辑模式：专注于编辑流程

### 2. 明确的操作入口
- "新建流程" → 进入空白画布
- "编辑" → 进入该流程的画布
- "返回列表" → 回到列表模式

### 3. 更大的编辑空间
- 编辑模式下画布占据整个内容区
- 更适合复杂流程的编排

### 4. 更好的性能
- 列表模式不加载画布
- 编辑模式不渲染列表
